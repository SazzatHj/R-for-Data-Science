## ১. আর-এ ভ্যারিয়েবল কী এবং এর গভীর মেকানিজম (Deep Dive into Variables)

### ভ্যারিয়েবল (Variable) কী?
সহজ ভাষায়, ভ্যারিয়েবল হলো একটি কম্পিউটারের মেমোরি বক্স বা পাত্রের মতো। ডেটা সায়েন্সে কাজ করার সময় আমাদের বিভিন্ন বিশাল ডেটাসেট, পরিসংখ্যানের ফলাফল বা টেক্সট সাময়িকভাবে কম্পিউটারের র‍্যামে (RAM) ধরে রাখতে হয়। যে নামের অধীনে আমরা এই ডেটাগুলো জমা রাখি, তাকেই **ভ্যারিয়েবল** বলে।

### অ্যাসাইনমেন্ট অপারেটর: কেন আর (R) ল্যাঙ্গুয়েজ অনন্য?
সাধারণত পাইথন বা সি-প্রোগ্রামিংয়ে ভ্যারিয়েবলে মান রাখার জন্য সমান চিহ্ন (`=`) ব্যবহার করা হয়। কিন্তু আর (R) ল্যাঙ্গুয়েজের অফিশিয়াল এবং আন্তর্জাতিক স্ট্যান্ডার্ড হলো **Arrow Operator (`<-`)** ব্যবহার করা।

* **কেন `<-` ব্যবহার করা হয়?** আর (R) মূলত ম্যাথমেটিক্যাল এবং স্ট্যাটিস্টিক্যাল কাজের জন্য তৈরি। এই চিহ্নের মানে হলো: *"ডান পাশের মানটিকে বাম পাশের কন্টেইনারের ভেতরে পুশ বা অ্যাসাইন করা হচ্ছে।"*

```r
### ❌ ব্যাড প্র্যাকটিস (প্রফেশনাল কোডাররা এটি এড়িয়ে চলেন)
sample_size = 500
```

### ভ্যারিয়েবল নামকরণের প্রফেশনাল নিয়মনীতি (Naming Conventions):
গিটহাবে আপনার কোড যখন বড় বড় প্রফেসরেরা দেখবেন, তখন আপনার ভ্যারিয়েবলের নাম দেখেই তারা আপনার দক্ষতার লেভেল বুঝে যাবেন। নামকরণের গোল্ডেন রুলসগুলো হলো:

* **Snake Case (স্নেক কেস):** একাধিক শব্দ থাকলে মাঝখানে আন্ডারস্কোর (_) দিতে হবে। যেমন: research_project_title <- "AI and Diplomacy"।

* **সংখ্যা দিয়ে শুরু নিষেধ:** আপনি data1 লিখতে পারবেন, কিন্তু 1data লিখতে পারবেন না। নামের শুরুতে কোনো সংখ্যা বসানো সম্পূর্ণ অবৈধ।

* **স্পেশাল ক্যারেক্টার নিষিদ্ধ:৮** নামের মাঝে স্পেস বা !, @, #, $, % এর মতো কোনো চিহ্ন ব্যবহার করা যাবে না।

* **কেস-সেন্সিটিভ (Case-Sensitive):** আর ল্যাঙ্গুয়েজে my_age এবং My_Age দুটি সম্পূর্ণ ভিন্ন ভ্যারিয়েবল।

## ২. আর-এর মূল ৫টি ডেটা টাইপ (Data Types in R with Real-World Context)
ডেটা সায়েন্স এবং গবেষণায় আমরা বিভিন্ন ধরনের ডেটা পাই। কোনোটি সংখ্যা, কোনোটি মানুষের নাম বা দেশের নাম, আবার কোনোটি হ্যা/না উত্তর। এই সবকিছুর ওপর ভিত্তি করে আর (R) ল্যাঙ্গুয়েজে প্রধানত ৫টি ডাটা টাইপ রয়েছে:

① Numeric (নিউমেরিক - দশমিক বা বাস্তব সংখ্যা)
এটি আর-এর ডিফল্ট সংখ্যার টাইপ। যেকোনো পূর্ণসংখ্যা কিংবা দশমিক সংখ্যাকে আর (R) সরাসরি নিউমেরিক হিসেবে গণ্য করে।

গবেষণায় ব্যবহার: জিডিপি (GDP), সিজিপিএ (CGPA), তাপমাত্রা ইত্যাদি স্টোর করতে।

```R
gpa_score <- 3.91
exchange_rate <- 117.50
```
② Integer (ইনটিজার - খাঁটি পূর্ণসংখ্যা)
যদি আপনি নিশ্চিত থাকেন যে আপনার ডেটাতে কোনো দশমিক থাকবে না (যেমন মানুষের সংখ্যা বা নমুনার সংখ্যা), তবে মেমোরি বাঁচানোর জন্য ইনটিজার ব্যবহার করা হয়। আর-কে বোঝানোর জন্য সংখ্যার শেষে একটি বড় হাতের L লিখতে হয় (L মানে Literal)।

গবেষণায় ব্যবহার: ইন্টারভিউ দেওয়া মানুষের সংখ্যা, দেশের সংখ্যা ইত্যাদি।

```R
total_respondents <- 150L
survey_year <- 2026L
```
③ Character (ক্যারেক্টার - টেক্সট বা স্ট্রিং)
যেকোনো ধরনের নাম, বাক্য বা বর্ণ যা ডাবল কোটেশন (" ") অথবা সিঙ্গেল কোটেশনের (' ') ভেতরে লেখা হয়।

গবেষণায় ব্যবহার: উত্তরদাতার নাম, দেশের নাম, থিসিসের টপিক ইত্যাদি।

```R
investigator_name <- "Sazzat Hj"
research_topic <- "Cybercrime Trends in South Asia"
```
④ Logical (লজিক্যাল - বুলিয়ান বা সত্য/মিথ্যা)
এটি শুধুমাত্র দুটি মান গ্রহণ করতে পারে: TRUE (সত্য) অথবা FALSE (মিথ্যা)। মনে রাখবেন, এগুলো সবসময় বড় হাতের অক্ষরে লিখতে হয়। (সংক্ষেপে T বা F ও লেখা যায়)।

গবেষণায় ব্যবহার: কোনো হাইপোথিসিস সত্য কি না, বা কোনো ডেটা মিসিং আছে কি না তা চেক করতে।

```R
has_cyber_cell <- TRUE
is_data_corrupted <- FALSE
```
⑤ Complex (কমপ্লেক্স - জটিল সংখ্যা)
বাস্তব সংখ্যার পাশাপাশি যখন কাল্পনিক অংশ (Imaginary unit i) যুক্ত থাকে। সাধারণ ডাটা সাইন্সে এর ব্যবহার খুব কম, তবে অ্যাডভান্সড ফিজিক্স বা গাণিতিক মডেলারদের জন্য এটি প্রয়োজন হয়।

```R
complex_number <- 5 + 3i
```
## ৩. টাইপ চেকিং এবং মেমোরি অ্যানালাইসিস (Checking Data Types)
কোডিং করার সময় মাঝেমধ্যে আমাদের নিশ্চিত হতে হয় যে একটি ভ্যারিয়েবল আসলে কোন ফর্মে আছে। এর জন্য দুটি অত্যন্ত পাওয়ারফুল ইন-বিল্ট ফাংশন রয়েছে:

১. class() ফাংশন (উচ্চ-স্তরের ক্যাটাগরি)
এটি ডেটার বাহ্যিক রূপ বা মূল ক্লাসটি আমাদের স্ক্রিনে দেখায়।

```R
status <- TRUE
class(status)
# 🖥️ Output: [1] "logical"
```
২. typeof() ফাংশন (অভ্যন্তরীণ মেমোরি রূপ)
এটি আর-এর ভেতরের মেমোরি쵝 আর্কিটেকচার লেভেলে ডেটাটি কীভাবে সেভ হয়েছে তা দেখায়। ব্যবহারিকভাবে class() তাকে "numeric" দেখালেও অভ্যন্তরীণভাবে typeof() তাকে "double" (দশমিক মেমোরি টাইপ) হিসেবে দেখায়।

```R
pi_value <- 3.1416
class(pi_value)   # 🖥️ Output: [1] "numeric"
typeof(pi_value)  # 🖥️ Output: [1] "double"
```
## ৪. ডেটা টাইপ রূপান্তর বা কাস্টিং (Data Type Conversion / Casting)
বাস্তব ডেটা অ্যানালাইসিসের সময় অনেক সময় সার্ভে বা এক্সেল ফাইল থেকে ডেটা লোড করার পর দেখা যায়, সংখ্যাগুলো টেক্সট (Character) হয়ে বসে আছে। সেই টেক্সট দিয়ে তো আর যোগ-বিয়োগ বা গ্রাফ তৈরি করা যাবে না। তাই ডেটা টাইপ চেঞ্জ করতে নিচের as. ফাংশনগুলো ব্যবহার করা হয়:

- as.numeric(): ক্যারেক্টার বা অন্য টাইপকে সংখ্যায় রূপান্তর করে।

- as.character(): সংখ্যা বা লজিক্যাল ডেটাকে টেক্সটে রূপান্তর করে।

- as.integer(): দশমিক সংখ্যাকে কেটে খাঁটি পূর্ণসংখ্যায় রূপান্তর করে।

```R
# বাস্তব সমস্যা: সংখ্যাটি টেক্সট আকারে আছে
wrong_budget <- "5000" 

# সমাধান: নিউমারিক-এ রূপান্তর
correct_budget <- as.numeric(wrong_budget)

# ভেরিফিকেশন
class(correct_budget)
# 🖥️ Output: [1] "numeric"
```

## ৫. এক্সপার্ট নোট ও ইন্টারভিউ টিপস (ভুল এড়ানোর কৌশল)
### Coercion (অটোমেটিক রূপান্তর) কী?
আর (R) ল্যাঙ্গুয়েজের একটি অদ্ভুত বৈশিষ্ট্য হলো, আপনি যদি একটি কন্টেইনারে ভিন্ন ভিন্ন ডেটা টাইপ মিক্স করেন, তবে আর (R) কাউকে না জানিয়ে নিজে নিজেই সব ডেটাকে সবচেয়ে শক্তিশালী ডেটা টাইপে রূপান্তর করে নেয়। একে Implicit Coercion বলে।

**শক্তি বা প্রাধান্যের ক্রমানুসারে:** Logical < Integer < Numeric < Character
অর্থাৎ, একটি টেক্সট এবং একটি সংখ্যাকে একসাথে মেলালে, পুরো জিনিসটি ক্যারেক্টার বা টেক্সট হয়ে যাবে!

```R
# উদাহরণ:
mixed_vector <- c(25, "Sazzat")
# এখানে আর (R) ভেতরের ২৫ সংখ্যাটিকেও "25" অর্থাৎ টেক্সট বানিয়ে ফেলবে!
```
**হ্যান্ডস-অন ল্যাব প্র্যাকটিস (Coding Practice Task)**
